# Async RL Design Folder

This folder is the dedicated home for Arcane's async RL and post-training design work.

These documents are intentionally grouped outside the general `docs/` plan area because this topic is expected to grow into a larger design stream with multiple iterations, specs, and implementation notes.

## Current documents

- `async_rl_architecture_discussion.md`: high-level architecture thesis and system framing
- `async_rl_library_implementation_review.md`: implementation-grounded review of cloned async RL libraries and the resulting Arcane design decisions
- `action_trace_discriminated_union_plan.md`: earlier lossless trace-events proposal that led into the broader async RL direction

## Intended future contents

- schema proposals
- interface specs
- rollout lifecycle notes
- trainer adapter contracts
- deployment topologies
- implementation roadmaps
