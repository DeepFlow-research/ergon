Walk the User Through a MAS Run

This walkthrough treats the run page as a visual debugger. The graph is a spatial aid, but time is the main axis. Details move into an inspector so the canvas stays readable.

Product thesis Do not show recursive subtasks all the way down by default. Show the event being explained, expand the affected branch, summarize hidden branches, and let the user scrub or pin context.
Moment 1

Landing: “Where is this sample right now?”

The default page opens at the latest meaningful moment. It shows current status, the active branch, and a compact timeline. The user should not need to pan around first.

Default expansion: root plus active branch only.
Hidden depth: collapsed branches show counts and status mix.
Primary action: scrub backward or click current task.

Moment 2

Scrub: “What changed at this point?”

Scrubbing does not just move a slider. It changes graph emphasis, opens the touched branch, and shows the state delta in plain language.

Selected event: one mutation or grouped event has ownership of the canvas.
Before/after: touched nodes show old → new status.
Noise control: unrelated branches dim or collapse.

Moment 3

Zoom out: “What chapter are we in?”

For post-run understanding, raw mutations are too granular. A phase map gives the user a stable story: created, delegated, executed, validated, evaluated.

Phase map: grouped by intent, not event type alone.
Drilldown: clicking a phase filters timeline and graph.
Audit: raw events remain available below the summary.
